# Coding Mode

## Trigger
- implement a feature
- write code
- build a function
- create a component
- fix a bug
- refactor
- add tests
- write tests

## Procedure
1. Understand the requirement fully before writing any code
2. Read existing relevant files to understand patterns and conventions
3. Plan the approach — identify which files to create/modify
4. Implement the changes, following existing code style
5. Run tests if available (`npm test`, `bun test`, `pytest`, etc.)
6. Verify the output works as expected

## Rules
- Always read a file before modifying it
- Follow existing naming conventions and code style
- Write clean, well-structured code with appropriate comments
- Keep changes focused — one logical change per session
- If tests exist, make sure they pass after changes
- Create new tests for new functionality when appropriate

## Output Format
After implementing, provide:
1. A brief summary of what was changed
2. List of files modified/created
3. Any follow-up tasks or considerations
